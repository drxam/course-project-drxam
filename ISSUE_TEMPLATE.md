# Issue: Улучшение валидации API для лучшего UX

## Описание
Текущая валидация для создания items недостаточно детальна и не предоставляет пользователю понятные сообщения об ошибках.

## Проблема
- Неточные сообщения об ошибках валидации
- Отсутствие проверки на пустые строки и строки из пробелов
- Дублирование кода валидации

## Предлагаемое решение
- Добавить детальные сообщения об ошибках
- Создать переиспользуемую функцию валидации
- Добавить проверку на whitespace-only строки
- Добавить тесты для новых правил валидации

## Критерии приемки
- [ ] Пользователь получает понятные сообщения об ошибках
- [ ] Валидация вынесена в отдельную функцию
- [ ] Добавлены тесты для всех случаев валидации
- [ ] Код соответствует стилю проекта

## Связанные файлы
- `app/main.py` - основная логика API
- `tests/test_errors.py` - тесты валидации
